# v0.1.0 - Initial public release

`llm-provider-audit` 的首次公开版本。

这个项目不是一个通用 benchmark，而是一个面向 provider 审计的 CLI 框架，用来回答更具体的问题：

> 某个 provider 实际返回的模型行为，是否真的接近它声称提供的模型？

## Highlights

- 支持 `inspect-router`、`calibrate`、`anchor-calibrate`、`run`、`report` 五个核心命令
- 使用 `reference calibration` 估计参考 provider 的自然波动，降低误判
- 支持 `negative controls` 与 `fingerprinting`，帮助区分“接近 official”与“同样接近低档模型”
- 支持本地 `codex-router` 隔离实例和任意 OpenAI-compatible provider
- 输出 `JSON + Markdown` 结果，便于归档、复核和后续自动化
- 自动加载 `.env`，同时提供可公开提交的模板配置

## Included public templates

- `configs/audits/openai-compatible-template.yaml`
- `configs/audits/openai-compatible-deep-template.yaml`
- `configs/audits/gpt-5.4-router-baseline.yaml`

## Current scope

- 重点是 provider 身份审计与模型替换检测
- 重点是可重复实验和结构化证据
- 当前还不是完整的历史巡检平台，后续会继续补 SQLite、Web 报告页和定时巡检能力
